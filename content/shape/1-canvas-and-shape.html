As we begin our journey into the field of graphic design, it's important to start at the very beginning. Graphic design is the art of using form and color to successfully convey a message, but this is not as easy as it sounds. The design process is one of trial and error, and it often takes many iterations to create a design that clearly communicates what you're trying to say. Although we tend to think of this design process as something involving gut instinct, experienced designers have an ever-expanding checklist of visual relationships that helps guide them through to the end.

The first item on this checklist is learning how to manipulate a single shape to produce desired outcomes. We will investigate how changing three simple variables of a rectangle – the position, size, and rotation – can result in drastically different expressions. Although basic, these variables are an important first set of techniques to be employed at the beginning of every design process. Design students are much better off focusing solely on getting these simple relationships right, instead of searching through online portfolios for inspiration. So let's look at the canvas, some simple shape variables, and how all of this relates to programming.

Size

The size of a shape can be used to communicate whether that shape is important or not. Thus, by increasing the size of important shapes and decreasing the size of less important shapes, you can bring clarity to a design, and this is a pattern we encounter daily in everything from traffic signs to website dialog boxes.

A famous example of sizing in graphic design is Helmut Krone and William Bernbach's Think Small advertising campaign for Volkswagen. In this series of magazine ads, an image of the Volkwagen Beetle is printed in different sizes to emphasize the supporting text copy.

You can reveal the underlying relationship of sizes by drawing black boxes on top of shapes in existing designs. This will highlight how sizing is an integral part of most successful designs, and how neglecting to think about sizing can lead to cluttered or unorganized layouts.

All basic drawing functions in P5.js allows you to specify the size of the shape you're drawing. To draw a rectangle, we use the rect(x, y, width, height) function. In the following example, we draw a rectangle that takes up half the width of canvas.

createCanvas(500, 500);
rect(0, 0, 250, 500);

Although this code certainly works, it doesn't really do a good job describing the relationship between the canvas and the rectangle. If I were to change the size of the canvas, the rectangle would no longer span half the canvas. The title of this book, Programming Design Systems, refers to the fact that when we create graphic designs in code, we are not merely drawing something, but programming a system to draw something. It is therefore up to us, the designers, to write our code so it is flexible, easy to understand, and easy to improve upon. In this case, you can make your code more flexible by calculating the size of your rectangle with the width and height variables. These variables are created for you by P5.js, and they will always hold the size of the canvas.

createCanvas(500, 500);
rect(0, 0, width/2, height);

The code above produces the same result, but it makes sure that the rectangle resizes based on the size of the canvas. This method is used throughout this book, as it's simply good practice, and it will make it easier for you to test variations of your code-based designs.

Position

Like words in a paragraph of text, the position of a shape indicates the order of the shape in the design. Most written languages have a natural reading direction, and this can be used to prioritize the importance of shapes, and tell the user where to start reading.

This pattern is used throughout the design world to create, in lack of a better word, rhythm. Graphic novels use the positioning of captions to guide the reader through the storyline. Newspapers place the most important text in a heading at the top of an article, which also tends to have a larger size to set it apart. Notice also how changing the position of shapes also changes the whitespace between them, which can be used to create both balance and asymmetry.

In these posters for Swiss Auto Club, Joseph Müller-Brockmann uses both position and size to prioritize certain shapes. On the left, the vehicle is emphasized to heighten the tension, making it apparent that the vehicle is on a collision course with the kid. On the right, the bicycle is emphasized to achieve the exact opposite.

To position 2D shapes in code, we use the concept of a coordinate system, where shapes are given x and y values that determine their position. To demonstrate best practice, take the following image.

We could easily implement this like the following code, where we correctly rely on the width and height variables to position the rectangle near the bottom right corner of the canvas.

rect(width - 150, height - 150, 100, 100);

However, even though the position of the rectangle is dynamic, the size is not. In the following example, we use the size of the canvas to calculate the size of the rectangle.

rect(width - (height*0.4), height - (height*0.4), height * 0.3, height * 0.3);

There is one big problem with the code above: It has become very hard to read. If you need to change the position of the rectangle, it will take you a while to mentally parse what's going on. To solve this problem, it is often helpful to use variables at the top of your code to store these numbers. This not only makes the code more readable, but it also makes it easier to test variations of your design. Rotation

Rotation can be used to make shapes come alive, as the whitespace around them becomes more complex and breaks the symmetry of the design.

Rotation is used in many types of design products: Children's books use rotated shapes to make them look fun and playful. Popular magazines use rotated grid systems to make the content look less dry, while publications for a narrow target audience use horizontal grid systems to imply quality of the content within.

In code, we rotate shapes by calling the rotate() function before drawing the shapes that need to be rotated. This function expects the rotation to be defined in radians, but it's often helpful to use the radians() function to convert degrees into radians. The following code will rotate all following shapes by 90 degrees.

rotate(radians(90));

It's important to understand that when you use the rotate() function, you are not rotating individual shapes. Instead, you are rotating the entire coordinate system, and because the shapes are drawn within the coordinate system, the shapes are rotated too. Combined with translate(), which moves the origin of the coordinate system, it's possible to achieve very different outcomes, as demonstrated in the following examples.

Designing a word

You can practice these techniques by picking an adjective from the dictionary and make a design for it by changing the position, size, and rotation of a single rectangle. This helps you build one of the most fundamental skills in graphic design: The ability to create visual relationships that make sense for your content.

As a simple example, let's take the word steep. How would you make a design for this word using just a rectangle and the three variables? Start by looking for connections between the word and your variables, and you'll notice that steepness and rotation are related: You can make the rectangle appear like a steep hill by rotating it. By changing the size, you can make that hill longer, continuing outside the canvas. By changing the position of the rectangle, you can create asymmetric whitespace, making sure that the user notices the rectangle.

Although this is just our first step into the world of graphic design, many extra-ordinary works of art have been created using this approach. It's also common to find problems in existing designs that can be traced back to the use of position, size, and rotation. Whether it's a user interface that fails to emphasize an important button, or a graph with ambiguity around its underlying dataset, it's clear that some designers rush to make things pretty without thinking about these basic relationships. This should be seen as a failure of design, and one is much better off including these ideas as a vital part of any design process.
